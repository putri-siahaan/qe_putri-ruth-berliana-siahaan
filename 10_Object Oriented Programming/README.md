# (10) Object Oriented Programming
Tiga poin yang dapat dipelajari dari materi terkait Object Oriented Programming yang diberikan:

## 1. Pengertian Object-Oriented Programming (OOP):
OOP adalah paradigma pemrograman yang berfokus pada pemodelan objek-objek yang memiliki atribut dan metode.
Dalam OOP, objek atau class membungkus data (atribut) dan perilaku (metode) menjadi satu kesatuan.

## 2. Mengapa Harus Menggunakan OOP:
OOP membantu meminimalkan kompleksitas dan meningkatkan keterbacaan kode.
Ini memungkinkan pengorganisasian yang lebih baik dari kode program, membuatnya lebih mudah dikelola dan dimodifikasi ketika program tumbuh.
OOP juga mendukung konsep pemrograman yang lebih abstrak, yang dapat membuat pemrograman lebih intuitif dan efisien.

## 3. Komponen OOP yang Penting:
- Class dan Object: Class adalah blueprint atau cetakan untuk membuat objek, sedangkan objek adalah instance konkret dari kelas tersebut.
- Atribut: Atribut adalah variabel yang menggambarkan karakteristik objek dalam sebuah kelas.
- Method: Method adalah fungsi atau tingkah laku yang dapat dilakukan oleh objek dalam suatu kelas.
- Inheritance (Warisan): Inheritance adalah hubungan antara kelas yang memungkinkan kelas anak (subclass) mewarisi atribut dan metode dari kelas induk (superclass).
- Modifier (Public, Private, Protected): Modifier mengatur tingkat akses ke atribut dan metode dalam sebuah class. public memungkinkan akses dari mana saja, private membatasi akses ke dalam class itu sendiri, dan protected memungkinkan akses dari class itu sendiri, subclass, dan package yang sama.
